Administration

Minecraft server

On the Minecraft server there are 3 types of users: Server, Operators (OPs) and Users.

Users

Everyone who joins the Minecraft server as a player will be a normal user. Their gamemode is by default adventure mode. This means that they cannot place or break blocks in the Minecraft world but they can interact with items such as buttons, signs, doors. They can also run the /goto command and cannot send hyperlinks in the chat

Operators

OPs have access to higher level commands for example they can: change the time/weather, add/remove OPs , teleport themselves/others, change gamemodes (giving them the ability to fly), kick and ban others. Operators can be set in the ops.json file in the backend directory. Simply look up the UUID of the Minecraft account you want to make OP Here. When the server starts the OPs are found from the ops.json file, since issuing the ops command does not add players to the ops.json file they will not persist through server restarts. So to make an OP persist add them to the ops.json file rather than using the ops command.

Server

The server is the highest level user. When running the server manually you are able to type commands into the shell form which the server is running. However since the gitlab runner starts the server you have to RCON into the server. RCON is a protocol that allows server administrators to remotely execute Minecraft commands as the server. This allows them to stop the server.

Mumble (Murmur) server

On the mumble server there are 4 types of users: Superuser, Admins, Authorised Users, Users.

Users

Everyone who joins the server is a user. As a user they will join the lobby channel but they will not have the ability to speak or hear or move channels. But they can register themselves.

Authorised (Registered) Users

Authorised users can speak to and hear others and can move channels. They have access to lobby and Minecraft A...C channels.

Admins

Authorised users can be made admins by other admins or by the Superuser. Admins are able to: create channels, kick/ban/unregister users, move users to different channels, and access the root channel.

Superuser

The Superuser is created along with the mumble server. The Superuser account has a password when joining the mumble server and is set during server setup. The Superuser has all of the functions of Admin but cannot speak.

Admin Page (for the lectures/quiz database)

Currently the admin page is not password protected so anyone on the imperial VPN can access this page. On this page you can insert lectures and quizzes into the database, these changes are reflected in real time on the minecraft server.
