Minecraft server

Introduction

This section will go through setting up a minecraft server in order to run the Minecraft virtual world. One of the key features if you intend to develop your own version of the world is the CI/CD pipeline that will make your workflow and continuous deployment much easier. The following examples apply to a Standalone Doc VM runningUbuntu 18.04.4 LTS (GNU/Linux 5.4.0-42-generic x86_64)

Setting up

First we need to setup some common tools on our server to support the minecraft environment.

Dev dependencies

Java

Install Java dependencies:
sudo apt install openjdk-8-jre-headless
sudo apt install openjdk-8-jdk-headless

Maven

Install Maven to build plugins:
sudo apt install maven

Git

Install GIT for version control:
sudo apt install git

Minecraft Dependencies

mcrcon

Installing mcrcon will give you control over the minecraft server from the outside of the running process. This is especially useful when we get to the CI/CD pipeline
You can follow this amazing tutorial to install mcrcon https://github.com/Tiiffi/mcrcon

Firewall set up

We will need to open two ports for the minecraft server: one for the world (25565), and one for remote control (25575). This can be achieve with:
sudo ufw allow 25565
sudo ufw allow 25575

Deployment

We will go through two different deployment approaches:

  • Cloning the git and running the world server as is without future integration. This will be useful if you intend to deploy multiple copies of the server on a cluster
  • CI/CD setup for continuous workflow in a development phase. This will be useful if you plan to carry on the integration of the project on a different server

Production deployment

This deployment does not require a pipeline setup and you simply need to run the server
