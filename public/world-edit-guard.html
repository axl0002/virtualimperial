1.3 Copy and Paste in Minecraft

Command Description
//copy The //copy command copies your current selection to your session’s clipboard, keeping track of where you are relative to the copy. The second part of that sentence is very important; if you want to later paste, for example, a bridge so that it is under where you are standing, you must stand in a location above the bridge when you make the copy. This method allows you to easily align your later paste because you can plan ahead a bit; it requires some spatial abilities to master the copying process but you will find it particularly helpful once you get the hang of it.
//paste Once you have something in your clipboard, you can paste it to the world. The last argument is optional: if you want the copy to paste at the same point that it was copied at, type //paste -o, otherwise the paste will be placed relative to you.

1.4 Moving things around

Command Arguments Description
//move <direction> Move every block in the selected region to a direction or based on the player's facing.

You point on the block of the extremity of the diagonal of what you want to select, type //hpos1, then point on the block on the other extremity of the diagonal and type //hpos2, and then you look in the direction you want to move it, and type //move 50

2. World Guard

2.1 Creating regions

All regions must have a name, and names must be unique. Names are case-insensitive. Create a region with /region define or its shortcut /rg define:

/region define town
/rg define town

The newly created region automatically prevents building within it. To let specific players build, add members or owners to the region. The main reason why you’d have separate owners or members is because you can set permissions to use commands separately for owners and members.

Either players or permission groups can be a member or owner. To specify a permission group, prefix the name of the group with g:. The relevant add and remove commands are illustrated below:

/rg addmember town Notch sk89q g:builders
/rg addowner town sk89q
/rg removemember town g:builders
/rg removeowner town sk89q

Full documentation for these commands can be found on the Region Commands page.

Example: Creating a spawn region where only the build team can build

  • Create a region named “spawn”:
    /rg define spawn
  • Add the build team as a member:
    /rg addmember spawn namesofmembers g:builders

2.2 Disable damages

We can change settings in spigot.yml including hunger, monster spawning and etc.

Some detailed settings like fall damage and pvp are not in spigot.yml:

/rg flag NAMEOFYOURREGION list
To see a list of settings you can change.
E.g. to turn off fall damage
/rg flag NAMEOFYOURREGION fall-damage deny
To turn off pvp damage
/rg flag NAMEOFYOURREGION pvp deny
